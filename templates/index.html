How to use the REST API

Go to /api/v1. There you will get the available endpoints. So far only the listing of all nametags, of single nametags and the creation of new nametags is implemented. Well, you can also retrieve the color map.

For example, try those out:

  • /api/v1/nametags - to retrieve all nametags that are in th store
  • /api/v1/nametags/1 - to retrieve the name tag with the ID 1
  • /api/v1/colormaps - to get (for now) the one color map available
  • /api/v1/templates ... we'll ... to experience witchcraft!

To create a new nametag you have to use a POST request to the /api/v1/nametags endpoint. As request body you have to provide a json object with at least a name and a pronouns value. Additionally you can use a gradient value which can be either true or false (which is the default, if not provided), a default_color and a fingerprint, both as hex values.

You can do this by using some application like Postman or doing a curl call from the command line like this: 

        curl -i -H "Content-Type: application/json" -X POST -d '{"name":"Frankie", "pronouns":"per, pers, pers", \
            "gradient":"true", "default_color":"aa00aa", "fingerprint":"ABCDEF0123456789ABCDEF0123456789" }'\
